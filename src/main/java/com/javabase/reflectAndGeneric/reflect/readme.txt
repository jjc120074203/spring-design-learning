通常情况下，程序对象类型 编译阶段都已经确定下来，加载成class对象 丢给 JVM，
JVM在创建对象之前会优先判断当前类是否加载，如果可以 在进行对象的内存分配 （new）各个对象之间只需要调用了就可以，
如果在运行期间 我这边需要加载一个类,但是JVM没有加载进去，我们让JVM不可能停止下来在对其进行修改吧。因此JVM 对此提供一个机制
可以在不停止程序的条件下，动态创建对象并且调用对象相关属性的方法-泛型

#反射的原理以及使用
反射 机制: 
java 在运行时，能够自动检查并且修改、获取自己内部的属性，以及变量。说白了java 语言在运行时期拥有的的一种自观查能力。

应用场景 ：IOC Dependency Jnjection(采用的这个方法)

反射提供：
Class ,Constructor,Field,Method,Modifier
